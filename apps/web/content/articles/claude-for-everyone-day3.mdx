# 第三天：多轮对话的艺术——让 Claude 越聊越懂你

前两天你学会了如何提问，掌握了 RTCF 模型。但在实际使用中，你会发现一个现象：**很少有人能一次就得到完美的回答。**

这不是你的问题，也不是 Claude 的问题。就像你和同事讨论方案一样，第一轮提出想法，第二轮补充细节，第三轮修正方向——好的结果往往是「聊出来」的。

今天我们来学习多轮对话的技巧，让你和 Claude 的协作效率翻倍。

每一轮对话，Claude 的「理解」都在加深。这就像和一个新同事合作——聊得越多，配合越默契。

## 多轮对话的4种核心策略

掌握以下4种策略，你就能在多轮对话中游刃有余：

### 策略1：渐进式细化

先给出大方向，再逐步补充细节。这适合你自己也不太确定最终要什么的场景。

```
第1轮：帮我写一篇关于远程办公的文章
第2轮：重点写远程办公对团队协作的影响
第3轮：加入3个具体的团队案例
第4轮：结尾加一段行动建议，面向管理者
```

### 策略2：反馈式修正

看到 Claude 的回答后，告诉它哪里好、哪里需要改。

```
第1轮：（初始请求）
第2轮：前两段写得很好，但第三段太正式了，改成口语化一点
第3轮：整体不错，但少了一个关键点——成本对比
```

### 策略3：角度切换

让 Claude 从不同的角度审视同一个问题。

```
第1轮：从用户的角度分析这个产品设计
第2轮：现在从开发者的角度看，有哪些实现难点？
第3轮：最后从老板的角度评估，值不值得做？
```

### 策略4：追问深入

当回答太笼统时，追问具体的细节。

```
第1轮：什么是敏捷开发？
第2轮：你说的 Sprint 具体是怎么运作的？
第3轮：能给我一个真实的 Sprint 计划例子吗？
```

## 什么时候该开新对话？

多轮对话虽然好，但不是每次都要在同一个对话里无限聊下去。以下情况建议开新对话：

- **话题完全切换时**：从写文章变成做数据分析，开新对话更干净
- **上下文太长时**：如果对话已经超过20轮，Claude 可能开始「遗忘」早期内容
- **需要新视角时**：同一个对话里 Claude 可能陷入某个思路，开新对话能获得全新的视角
- **出现重复或质量下降时**：当你发现回答开始重复或质量变差，这是上下文窗口拥挤的信号

## 实战练习

今天的练习需要你用至少3轮对话来完成一个任务：

**任务：让 Claude 帮你写一段自我介绍**

```
第1轮：帮我写一段面试用的自我介绍，我是[你的真实情况]
第2轮：（看完回答后）开头太平了，能不能用一个有趣的故事开头？
第3轮：（看完修改后）整体不错，但能不能控制在1分钟内说完？大约200字。
```

体验完这个过程，你会真正理解多轮对话的威力。

## 今日小结

1. **多轮对话 > 单轮提问**：好的结果是「聊出来的」，不要期望一步到位
2. **4种核心策略**：渐进式细化、反馈式修正、角度切换、追问深入
3. **知道何时开新对话**：话题切换、上下文过长、需要新视角时
