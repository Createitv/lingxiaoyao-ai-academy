经过前两周的学习，你已经掌握了和 Claude 高效对话的各种技巧。但你可能遇到过一个问题：每次开新对话都要重新解释背景，之前上传的文件也找不到了。今天我们来解决这个痛点——**Claude Projects**。

## 什么是 Claude Projects？

Claude Projects 是 Anthropic 在 Claude Pro 和 Team 版本中提供的项目管理功能。它让你可以为不同的工作场景创建独立的工作空间，每个空间都有自己的知识库和系统设定。

### 步骤详解

**第一步：创建项目。** 在 Claude 界面左侧栏，点击"Projects"区域，然后选择"Create a Project"。给项目起一个清晰的名字，比如"市场调研助手"或"周报撰写"。

**第二步：上传知识库。** 这是 Projects 最强大的功能。你可以上传公司文档、产品手册、行业报告、代码库等任何参考资料。Claude 会在后续对话中随时引用这些内容。支持 PDF、文本文件、代码文件等多种格式，单个项目最多可上传约 200,000 个 token 的内容。

**第三步：编写系统提示词。** 在项目设置中，你可以为 Claude 定义角色和行为规则。例如：

```
你是一位资深的市场分析师。请始终基于项目知识库中的数据来回答问题。
回答格式要求：先给结论，再给论据，最后给建议。
如果知识库中没有相关信息，请明确告知用户。
```

**第四步：开始对话。** 在项目中新建对话，Claude 就会自动加载你设定的系统提示词和知识库，无需每次重新说明背景。

**关于知识库的小技巧**：上传文档时，建议在文档开头加一段摘要，告诉 Claude 这个文档是关于什么的。这样 Claude 在检索信息时效率更高。另外，如果你的知识库涉及专业术语，可以在系统提示词中加上术语表，确保 Claude 的理解和你一致。

## 系统提示词的编写技巧

系统提示词是 Projects 的灵魂。一个写得好的系统提示词，能让 Claude 在每次对话中都保持一致的高质量输出。以下是编写系统提示词的实用建议：

**明确角色定位**：不要只说"你是一个助手"，而要说"你是一位拥有5年经验的科技媒体编辑，熟悉人工智能、云计算和物联网领域"。角色越具体，输出越精准。

**设定行为规范**：告诉 Claude 在这个项目中应该怎么做、不该怎么做。比如"回答问题时必须引用知识库中的具体文档"或"不要使用过于学术化的语言"。

**规定输出格式**：如果你需要固定格式的输出，在系统提示词中明确说明。比如"每次回答都先给一句话结论，再展开分析，最后列出行动建议"。

**加入质量标准**：设定你对输出质量的期望，比如"如果无法从知识库中找到答案，请明确说明而不是猜测"。

## 实操建议

今天的练习任务：创建你的第一个 Claude Project。

1. 想一个你经常需要 Claude 帮忙的场景（比如写周报、整理会议纪要）
2. 准备 2-3 份相关文档（之前的周报、会议模板等）
3. 编写一段系统提示词，定义 Claude 在这个场景中的角色和行为规范
4. 在项目中开始一轮对话，测试效果
5. 根据测试结果，调整系统提示词并迭代优化

你会发现，有了 Projects，Claude 就像一个真正了解你工作背景的同事，沟通效率大幅提升。而且随着你不断优化系统提示词和丰富知识库，这个"虚拟同事"会越来越懂你。
