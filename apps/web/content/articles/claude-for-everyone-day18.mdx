# 第十八天：打造你的Prompt模板库——提高效率的秘诀

你有没有发现，很多时候你问 Claude 的问题其实大同小异？写邮件、做总结、分析数据……每次都要重新组织提示词，既费时又容易遗漏关键要素。今天我们来解决这个问题——建立你自己的 **Prompt 模板库**。

## 为什么需要模板库？

一个好的 Prompt 模板库能帮你：
- **节省时间**：常用场景一键调用，不用每次重新想提示词
- **保证质量**：经过反复优化的模板，输出质量更稳定
- **团队共享**：好的模板可以分享给同事，整个团队都受益
- **持续进化**：模板不是一成不变的，每次使用都可以迭代改进

### 模板示例：商务邮件

```
你是一位经验丰富的商务沟通专家，擅长写正式但不生硬的中文邮件。

请帮我写一封邮件：
- 收件人：[收件人职位和姓名]
- 目的：[邮件目的]
- 关键要点：[1-3个要传达的核心信息]
- 语气：[专业正式 / 友好亲切 / 紧急重要]
- 字数：控制在[200/300/500]字以内

要求：开头称呼得体，正文条理清晰，结尾有明确的行动号召。
```

### 模板示例：内容总结

```
请阅读以下内容，并按要求总结：

[粘贴原文内容]

总结要求：
1. 一句话核心观点（不超过30字）
2. 3-5个关键要点（每点一句话）
3. 对[目标读者]的实际意义
4. 如果有数据，请保留关键数字

输出格式：Markdown，使用标题和要点列表。
```

## 模板迭代方法

模板不是一次写好就完事了。每次使用后，记录什么地方效果好、什么地方需要改进。推荐一个简单的迭代流程：

1. **首次使用**：记录 Claude 的输出质量（1-5 分）
2. **发现问题**：输出太长？格式不对？遗漏要素？
3. **调整模板**：针对问题修改角色描述、任务细节或约束条件
4. **再次测试**：验证修改是否有效
5. **固化版本**：效果满意后标注版本号（V1、V2...）

建议用 Notion、飞书文档或简单的 Markdown 文件来管理你的模板库。每个模板记录：模板名称、适用场景、最新版本、上次更新日期。

**关于模板的共享**：如果你在团队中工作，强烈建议把优质模板共享给同事。你可以建立一个团队模板库，每个人贡献自己领域的最佳模板，整个团队的 AI 使用水平都会提升。好的模板是一种知识资产，越分享越有价值。

## 实操任务

今天的练习：创建你的第一个模板。

1. 选一个你每周至少用 2 次以上的场景
2. 按照 [角色] + [任务] + [约束] 的结构写一个模板
3. 标注出需要每次填写的变量（用方括号）
4. 用实际内容测试一次
5. 根据输出质量做第一次迭代

当你积累到 10 个以上高质量模板时，你会感受到效率的飞跃。
