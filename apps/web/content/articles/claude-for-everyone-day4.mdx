# 第四天：让 Claude 帮你读文件——文档分析实战

前三天你学会了和 Claude 对话的基本功。从今天开始，我们进入更实用的技能——**让 Claude 帮你处理文件**。

想象这些场景：一份50页的合同需要审查、一篇英文论文要快速了解核心观点、一段代码要检查有没有 bug。以前你可能要花几个小时逐字阅读，现在只需要把文件丢给 Claude，几秒钟就能得到分析结果。

此外还支持纯文本文件（.txt）、Markdown 文件（.md）、HTML 文件等。基本上你日常工作中会遇到的文件类型，Claude 都能读。

**一个重要限制**：Claude 目前对单个文件的处理能力大约在数万字左右。如果你的文件特别大（比如超过200页的 PDF），建议分段上传。

## 三大实战场景

### 场景1：合同审查

合同审查是 Claude 文档分析最受欢迎的用途之一。以下是一个实用的 Prompt 模板：

```
请以法律顾问的视角审阅这份合同，重点关注：
1. 对我方不利的条款有哪些？
2. 有没有不合理的违约金或处罚条款？
3. 终止条款是否合理？
4. 有没有模糊不清、可能产生歧义的表述？
请按严重程度从高到低排列。
```

### 场景2：报告总结

当你面对一份冗长的报告时：

```
请阅读这份报告，完成以下任务：
1. 用200字以内总结核心观点
2. 列出报告中最重要的3个数据/发现
3. 指出报告的结论和建议
4. 这份报告有没有明显的逻辑漏洞？
```

### 场景3：代码审查

开发者可以让 Claude 帮忙审查代码：

1. 有没有潜在的 bug 或安全问题？
2. 代码的可读性和可维护性如何？
3. 有没有性能优化的空间？
4. 建议如何重构？

## 文档分析的5个技巧

1. **先让 Claude 概述，再深入细节**：上传文件后，先问「请用5句话概述这份文件的主要内容」，了解全貌后再深入特定部分

2. **指定分析角度**：「从财务角度看」「从用户体验角度看」「从法律风险角度看」——不同角度会得到截然不同的分析

3. **要求引用原文**：「请引用文件中的原文来支持你的分析」——这样你可以验证 Claude 的分析是否准确

4. **对比分析**：上传两份文件，让 Claude 做对比。比如两个版本的合同、两家公司的报告

5. **注意 Claude 的局限**：Claude 不能执行文件中的代码，不能修改原文件。它是「读」和「分析」，不是「编辑」。分析完成后的修改工作需要你自己完成

## 重要注意事项

使用 Claude 分析文件时，请注意以下几点：

- **隐私安全**：不要上传包含极度敏感信息（如银行密码、身份证号）的文件
- **准确性验证**：Claude 的分析是辅助参考，重要决策（如签合同）仍需专业人士确认
- **文件大小**：如果文件太大，可以截取关键部分上传
- **语言混合**：Claude 支持中英文混合的文件，你可以用中文提问英文文件

## 实战练习

找一份你手边的真实文件（工作文档、一份合同、一份报告），上传给 Claude，尝试：

1. 先让它做一个整体概述
2. 再针对你最关心的部分深入提问
3. 最后让它生成一份分析摘要

## 今日小结

1. **Claude 支持多种文件类型**：PDF、Word、Excel、代码、图片等主流格式都可以处理
2. **文档分析三步走**：上传文件 → 精准提问 → 获取洞察
3. **问对问题是关键**：指定分析角度、要求引用原文、先概述再深入
