# 第十二天：让输出格式为你所用

你有没有遇到过这种情况：Claude 的回答内容很好，但格式不对——你想要表格它给了你文字，你想要列表它给了你段落？今天我们来解决这个问题，学会让 Claude 精确地按你要求的格式输出。

## 为什么格式很重要

同样的信息，不同的格式服务于不同的目的。纯文本适合阅读，表格适合对比，JSON 适合系统间传递数据，Markdown 适合生成文档。选对格式，信息的价值能翻倍。

## 如何指定输出格式

让 Claude 按特定格式输出，关键在于**明确告诉它你要什么**。以下是一些实用的格式指定方法。

### 表格输出

```
请用Markdown表格对比以下三款手机的参数：iPhone 16 Pro、华为Mate 70 Pro、三星Galaxy S25 Ultra

表格列：品牌型号、处理器、屏幕尺寸、电池容量、起售价、主要卖点
```

Claude 会输出整齐的 Markdown 表格，你可以直接复制粘贴到文档或笔记中。

### JSON 输出

当你需要结构化的数据用于其他工具时，JSON 格式最合适：

```
请将以下产品信息整理为JSON格式：
- 产品名：智能台灯
- 价格：299元
- 颜色：白色、黑色、木纹
- 功能：调光、定时、语音控制
- 重量：1.2kg

要求：字段名使用英文驼峰命名
```

### Markdown 文档

```
请用Markdown格式写一份产品需求文档，包含：
- 一级标题：产品名称
- 二级标题：需求背景、目标用户、核心功能、非功能需求
- 核心功能部分用有序列表
- 重点内容用加粗标注
```

## 高级技巧：模板输出

如果你经常需要相同格式的输出，可以给 Claude 定义一个模板：

```
请按照以下模板为每个团队成员生成月度绩效评语：

【姓名】：___
【岗位】：___
【本月亮点】：（1-2句话描述突出表现）
【改进建议】：（1-2句话描述改进方向）
【综合评级】：A/B/C/D
【主管建议】：（一句话总结）

需要评价的成员：张三（设计师）、李四（前端开发）、王五（产品经理）
```

这种模板化的方式特别适合批量生成内容，格式高度统一。

**场景一：会议纪要**

```
请将以下会议录音文字稿整理为会议纪要，格式要求：
- 会议主题
- 参会人员
- 讨论要点（编号列表）
- 决议事项（表格：事项/负责人/截止日期）
- 待跟进问题
```

**场景二：竞品分析**

```
请将以下竞品信息整理为JSON数组，每个竞品包含：
name（名称）、price（价格）、features（功能列表）、
pros（优点）、cons（缺点）、rating（评分1-10）
```

**场景三：技术文档**

```
请用Markdown格式编写API文档，包含：
## 接口名称
### 请求方式
### 请求参数（表格）
### 返回示例（代码块）
### 错误码说明（表格）
```

## 格式转换技巧

Claude 还是一个出色的格式转换器。你可以让它在不同格式之间互相转换：

- 「把这段 JSON 转成 Markdown 表格」
- 「把这个表格数据转成 CSV 格式」
- 「把这份会议纪要从纯文本转成带格式的 Markdown」
- 「把这个 HTML 表格提取为 JSON 数据」

## 今日小结

掌握结构化输出是提升 Claude 使用效率的关键一步。记住三个要点：**明确告知格式名称**、**给出示例或模板**、**指定字段和约束**。当你养成了指定输出格式的习惯，Claude 就能成为你真正的文档工厂。
