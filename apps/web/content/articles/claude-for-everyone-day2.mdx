# 第二天：学会提问——掌握 Prompt 的4个核心要素

昨天你完成了和 Claude 的第一次对话，感受到了 AI 的能力。但你可能也注意到了一个问题：有时候 Claude 的回答并不完全是你想要的。

这不是 Claude 不够聪明，而是我们的「提问方式」还不够好。

今天我们来解决这个问题。你将学会一个简单但极其有效的框架——**RTCF 模型**，让你的每一次提问都能精准命中。

## 什么是 Prompt？

Prompt（提示词）就是你输入给 Claude 的那段话。它不是什么神秘的咒语，本质上就是一种**沟通方式**。

想象一下，你去餐厅点菜。如果你只说「来点吃的」，服务员会很困惑；但如果你说「来一份微辣的水煮鱼，米饭一碗，少放盐」，你就能得到想要的东西。

和 Claude 对话也是一样。提问越清晰、越具体，回答就越接近你的期望。

### R — Role（角色）

告诉 Claude 它应该扮演什么角色。这会直接影响它的语气、专业度和视角。

例如：「你是一位有10年经验的市场营销专家」「你是一位耐心的小学语文老师」「你是一位严谨的法律顾问」

### T — Task（任务）

明确告诉 Claude 你需要它做什么。任务描述越具体越好，避免模糊的表述。

例如：「帮我写一封拒绝加班的邮件」而不是「帮我写封邮件」

### C — Context（背景）

提供必要的背景信息，帮助 Claude 理解你的具体情况。没有背景，Claude 只能给出通用回答。

例如：「我是一个3人创业团队的产品经理，公司做的是在线教育产品，目前还没有盈利」

### F — Format（格式）

告诉 Claude 你希望结果以什么样的格式呈现。

例如：「用表格形式对比」「分为3个部分，每部分100字左右」「用要点列表，不超过5条」

**重要提醒**：不是每次提问都需要用满4个要素。简单的问题，只用 T（任务）就够了。RTCF 是一个「检查清单」，不是「必填表格」。

## 实战练习

现在轮到你来练习了。请打开 Claude，尝试以下3个场景：

**场景1：请假邮件**

用 RTCF 模型构建你的 Prompt：
```
你是一位职场沟通专家（R）。帮我写一封请假邮件（T）。
我在一家互联网公司做设计师，因为周五需要去医院复查，想请一天假。
我的直属领导叫张总，他比较注重工作交接（C）。
邮件要简短专业，不超过150字（F）。
```

**场景2：学习计划**

```
你是一位学习规划师（R）。帮我制定一个英语口语提升计划（T）。
我目前英语大约四级水平，每天能抽出30分钟学习，目标是3个月后能进行基本商务对话（C）。
请按周为单位列出计划，用表格格式（F）。
```

**场景3：自由发挥**

想一个你工作或生活中的真实场景，用 RTCF 模型构建 Prompt，看看效果如何。

## 今日小结

1. **Prompt 就是沟通**：你提供的信息越清晰，Claude 的回答越好
2. **RTCF 模型**：角色（Role）、任务（Task）、背景（Context）、格式（Format）——四个要素让提问变得系统化
3. **灵活运用**：不必每次都用满4个要素，根据需求灵活组合
